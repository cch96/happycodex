# HappyCodex 0.3 ExecPlan

## When to create it

Before research/challenger dispatch, persist a qualifying skeleton at the repository policy path or `docs/execplans/<task-slug>.md`. Native Plan is cursor; Goal explicit.

## Template

Protocol: `HappyCodex/0.3`
Invocation: `$happycodex:happycodex`
Writer: Root only
State: skeleton / frozen / active / closed
Resume: read this file and reconcile native facts before any write, review, or completion.

### Outcome and baseline

**Operative sources.** Append verbatim user text with anchor/hash; only later verbatim authorization supersedes it. Goal/Outcome divergence stops; close Goal after every gate. A waiver needs explicit user text, named gate, exact scope, reopen condition, and unrun-gate disclosure.

**Amendment.** Change/reason: `<delta>`; Authorization anchor/hash: `<source/digest>`; Affected claims: `<IDs>`; Invalidated evidence: `<IDs/digests>`; Required reruns: `<commands/gates>`.

**Normalized outcome.** Map every source clause to success, preservation, allowed breaks, exclusions, and stops.

**Design saturation.** Record decision-changing options, omission challenge, boundary/rejections, and saturation; omit cosmetic variants.

**Immutable snapshot.** Repository: `<name>`; revision/tree: `<full IDs>`; worktree/status: `<path/status>`; owner: `Root/preserved`; operative baseline: `yes/no`.

Advance only with quoted authorization; version, rerun, and reopen evidence.

**Baseline check.** Command: `<exact>`; exit: `<code>`; structured failure identities/count: `<IDs/signatures>`; normalized digest: `<digest>`; accepted: `yes/no`.

Freeze checks; candidate failures are not baseline and unaccepted failures stay open. Sanitize sensitive logs.

### Claims Ledger

Use one table. Types are `outcome`, `preservation`, `allowed-break`, and `premise`; states are only `open`, `verified`, or `N/A`; `N/A` requires path, search, or command evidence.

The contract-freeze revision freezes the user contract: operative sources, normalized Outcome, exclusions, allowed breaks, and baseline. Monotonically add a derived claim without new authorization only when entailed by a frozen clause or baseline-reachable behavior; record provenance and reopen evidence. Send anything else to the external backlog. Removing or narrowing, widening a break, `N/A`, or changing Outcome/baseline requires explicit user authorization.

### Retrospective

Fill only after final completion; record lessons/risks, do not expose it to a reviewer, and never use it as evidence.

## Neutral review brief

Phase 1 receives verbatim requests/amendments, source and synthetic commits, scope, accepted baseline failures, objective facts, and exclusions. Exclude the ExecPlan; prove manifest equality. Before candidate evidence, freeze numbered independent obligations, then inspect correctness/compatibility and every obligation/diff unit. Missing inventory permits one same-session correction; incomplete correctness stops.

Only then send contract-only sources, Outcome, claims, and oracles; exclude evidence, decisions, self-review, findings, receipts, Retrospective, repair story, and verdict.

## Coverage receipt

Record reviewer/session/command/durable output; configured and effective model/effort/permissions; authorization/degradation; source/synthetic commits and manifest; every diff unit and obligation; queries/counts/paths/evidence/limits/mapping; external reads; contamination/truncation/omissions; Root dispositions and rejected counter-evidence. Send noncontract issues to the external backlog. Keep findings in native output; persist only metadata/digests/counts/scope/limitations.

Append receipt/closure administratively; prove product-tree identity excluding only the ExecPlan. Product/claim changes reopen evidence.
